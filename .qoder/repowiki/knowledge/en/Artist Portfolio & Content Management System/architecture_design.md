- **Entry & Routing**: `main.tsx` bootstraps the app with global providers (`AuthProvider`, `ContentProvider`). `App.tsx` defines client-side routes using `react-router-dom`, separating public pages (e.g., `Index`, `Music`) from protected admin routes guarded by `RequireAdmin`.
- **State Management**: Global state is managed via React Context. `AuthContext` handles session persistence in `localStorage`, while `ContentContext` fetches and caches site content (albums, tours, hero data) from the backend API using `@tanstack/react-query` patterns.
- **Data Layer**: `lib/api.ts` provides a centralized `adminApi` client for RESTful interactions with the backend, handling JSON serialization and error propagation.
- **UI Composition**: The UI is built on a component-based architecture using `shadcn/ui` primitives in `components/ui/`. Complex views like `Hero.tsx` integrate third-party services (YouTube iframe API) and local state for interactive features like search and video controls.
- **Admin Interface**: Admin pages in `pages/admin/` delegate editing logic to large composite components in `components/admin/AdminDashboard.tsx`, which manage form state and trigger API updates for specific content sections.
